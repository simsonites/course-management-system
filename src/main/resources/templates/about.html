Course Registration Management (CRM) Overview

CRM is a simple course registration management system where students can go and register for maximum of three (3) to offer for a semester.

How it works

CRM consist of two roles, The STUDENT ROLE and THE ADMIN ROLE. The Admin panel is restricted to only users that login with ADMIN ROLE

What can a student do ?

A student can:

  • Create account.
  • Register for courses.
  • Update their profile
  • Add and Remove courses to/from their profile
  • Delete their profile
  • Browse courses.

What can Admin user do ?

Admin can :

  • Create account.
  • Manage courses: (Perform full CRUD operation)
  • Manage students (users) (Perform full CRUD operation)
  • Add and remove Admins.

Technologies Used

  • Java.
  • MySQL Database.
  • Hibernate (ORM)
  • Spring Frameworks (MVC, DATA-JPA, BOOT, SECURITY)
  • Maven dependency management
  • Thymeleaf view
  • Bootstrap 4 (CSS)
  • GitHub (source control)
  • App hosted on AWS

What more could be done in future?

This app does not contain all fuctionalities that a standard CRM should have. But it has all basic CRUD AND SECURITY functionalities as a building block for the addition of more advanced functionalites as per requirement. So this is a demonstration of fully fledged working application built from scratch by the aforementioned set of technologies.
